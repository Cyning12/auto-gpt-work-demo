# 运维助手可调用的工具说明（知识库 mock）

本文档供智能助手在回答用户前查阅，明确**各工具职责与调用时机**。  
（课程练习数据，非真实生产配置。）

---

## 1. `get_current_status`（监控系统快照）

**作用**：向本练习中的**模拟监控接口**拉取**当前时刻**的数据库服务器指标。

**返回字段（JSON）**：

- `连接数`：当前活跃连接数（整数）
- `CPU使用率`：百分比字符串，如 `"62.3%"`
- `内存使用率`：百分比字符串

**何时应调用**：

- 用户**明确**询问「现在 / 当前 / 此刻」的连接数、CPU、内存等**实时数值**；
- 已在文档（如《服务器异常处置手册》）中给出处置步骤后，需要**补充一份当前指标**辅助判断（例如连接数是否仍处于高位）。

**何时不应单独依赖本工具**：

- 用户问的是**处理步骤、SOP 条款、联系人、制度规定、备份时间、报销标准**等——必须先查阅对应**文本文档**（通过 `list_doc_files` + `read_local_file`），**不能**仅用本工具代替书面流程。

---

## 2. `list_doc_files` + `read_local_file`（知识库文档）

**作用**：枚举 `doc` 目录下可读文件，并按路径读取**完整文本**（制度、SOP、政策、规范等）。

**典型文件**：

- `服务器异常处置手册 (SOP).txt` — 告警与数据库异常处置步骤
- `DB_Backup_Policy.md` — 备份与恢复策略
- `员工差旅报销制度 (Policy).txt` — 差旅报销标准与时限
- `Frontend_Review_Guide.txt` — 前端 Code Review 要求
- `Q1_Teambuilding.txt` — 团建安排
- `Seattle_Hotels.csv` — 示例表格数据（按需读取）

**何时必须先调用**：凡涉及「按公司规定、手册、制度、步骤、联系人」的回答，均应**先列表再读文件**，再组织结论。

---

## 3. 推荐组合顺序（示例）

1. 用户：**「连接数告警，按手册第一步该做什么？」**  
   → 先 `list_doc_files`，再 `read_local_file` 打开 `服务器异常处置手册 (SOP).txt`，据文作答。  
   → 若用户还要求「现在连接数多少」，再调用 `get_current_status`。

2. 用户：**「现在 CPU 多少？」**  
   → 直接 `get_current_status` 即可。
