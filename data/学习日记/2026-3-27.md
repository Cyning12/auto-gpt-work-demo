📘 核心知识点提取与深度拆解
1. RAG 的核心原理：检索 + 生成 (Retrieval + Generation)
知识点说明：RAG 是一种通过从外部知识库检索相关信息，并将其作为上下文（Context）输入给大模型（LLM），以增强生成结果准确性的技术。

关联实操：你之前把《三国演义》切片存入 FAISS，就是在构建“检索”部分；将搜到的 Snippet 发给 LLM，就是在完成“生成”部分。

2. Native RAG 的标准流程
知识点说明：包括数据准备（Load, Split, Embedding, Vector Store）、检索（Similarity Search）和生成（Prompt + Context）。

关键参数：

Chunk Size (分片大小)：决定了信息的粒度。

Chunk Overlap (分片重叠度)：保证语义连贯性，防止信息在切分处断裂。

关联实操：你将分片从 3500 改为 500，就是对 Chunk Size 的调优。

3. 常见问题与质量提升（RAG 的“救命稻草”）
语义偏移：当 Embedding 匹配度低时，会搜到不相关的片段（比如你搜到的姜维）。

幻觉处理：模型在检索不到信息时可能会胡编。

提升方案：

多路召回 (Hybrid Search)：结合关键词和向量。

重排序 (Rerank)：在初步搜到的 Top-N 中进行更精细的二次筛选。

Query 改写：让 LLM 先把用户的问题转化成更适合检索的表达方式。

4. 动态防护栏 (Guardrails)
知识点说明：在答案生成阶段设置规则，验证内容是否与检索到的知识一致，防止幻觉。

关联实操：这正是你昨天提到的“怎么判断是否该用插件”的高级版。
