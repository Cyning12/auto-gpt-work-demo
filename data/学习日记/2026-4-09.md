## 二、核心实现思路（从“跑通”到“稳定”）

### 1）统一调用与解析：避免“输出一行”把 JSON 截断

早期问题：`auto_rewrite_query()` 让模型输出多行 JSON，但如果把输出“清洗为第一行”，就会把 JSON 截成 `{`，导致 `json.loads` 必然失败，从而触发兜底返回（`未知类型/0.5`）。

现在的实现方式：

- `_call_rewrite_model(system, user, output_format=...)` 做统一入口  
- `output_format` 支持：
  - **`one_line_text`**：只适合“改写成一句话”的任务
  - **`json_object`**：解析 JSON 对象（用于类型识别）
  - **`json_array`**：解析 JSON 数组（用于多意图拆分）
- `_extract_first_json_object()` / `_extract_first_json_array()`：容错提取（支持多行、少量噪声、甚至被 ```json``` 包裹）

### 2）策略方法：按“输出形态”选择解析方式

- `rewrite_context_dependent_query()`：输出一行“独立问句”
- `rewrite_ambiguous_reference_query()`：输出一行“去指代问句”
- `rewrite_comparative_query()`：输出一行“明确对比对象的问句”
- `rewrite_multi_intent_query()`：输出 **JSON 数组**（多个子问题），返回 `list[str]`
- `rewrite_rhetorical_query()`：输出一行“中立客观问句”（强调**不要分析**，避免返回 `**分析：**` 这类首行污染）

## 三、踩过的坑与修复

### 1）导入路径坑：从子目录运行找不到公共模块

- **现象**：`python query_rewrite.py` 报 `No module named 'dashscope_generation'`
- **根因**：公共模块在 `data/课程练习/`，运行目录在 `RAG技术与应用/`
- **修复**：与其它练习脚本一致，注入 `_PRACTICE_ROOT` 到 `sys.path`

### 2）JSON 解析坑：清洗“第一行”破坏结构化输出

- **现象**：模型明明返回 JSON，但程序总是走兜底 `未知类型/0.5`
- **根因**：把多行 JSON 清洗为一行后只剩 `{`，`json.loads` 必失败
- **修复**：把“文本改写”和“结构化 JSON 输出”分开解析通道（`output_format`）

### 3）反问改写坑：模型先输出分析导致首行污染

- **现象**：改写结果变成 `**分析：**`
- **根因**：提示词没强约束“只输出一行改写结果”
- **修复**：在 `rewrite_rhetorical_query()` 明确规则：只输出一行中立问句，不要分析/解释

## 四、Demo 的使用方式

### 1）直接运行（内置 case1~case6）

```bash
cd data/课程练习/RAG技术与应用
python query_rewrite.py
```

### 2）在代码里调用（示例）

```python
from query_rewrite import QueryRewrite

qr = QueryRewrite()
res = qr.auto_rewrite_and_execute("我想了解门票价格、有哪些必玩项目？以及怎么去？")
print(res["rewritten_query"])  # 多意图：list[str]
```
